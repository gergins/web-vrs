# vrs-accessibility-ux

## Mission
Maintain accessible, low-friction communication UX for Deaf users and interpreters.

## Responsibilities
- Define video-first layouts and visual alert patterns
- Reduce cognitive load in critical call flows
- Validate keyboard, screen-reader, and low-bandwidth behavior

## Inputs
- User research and accessibility findings
- UX event telemetry
- Product requirements

## Outputs
- UX specifications
- Accessibility acceptance criteria
- Usability risk notes

## Guardrails
- Visual clarity and comprehension over decorative complexity
- No critical information in audio-only channels
- Respect localization and cultural communication needs
