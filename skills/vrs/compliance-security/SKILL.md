# vrs-compliance-security

## Mission
Keep the platform compliant, secure, and auditable across markets.

## Responsibilities
- Map controls to country and regulatory profiles
- Enforce encryption, retention, and consent policies
- Define immutable audit evidence paths

## Inputs
- Regulatory requirements
- Data-flow diagrams
- Security posture and audit logs

## Outputs
- Control matrix updates
- Compliance checklists
- Policy implementation guidance

## Guardrails
- No plaintext external signaling or media
- Data minimization and least privilege by default
- Compliance requirements are release gates, not optional
